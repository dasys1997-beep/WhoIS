# BookMind

Telegram Mini App для читачів детективів та інших книг з великою кількістю персонажів.

## Версія 1.5 — уточнення після тестування

**Свайп тепер одноступеневий** — без проміжної фіксації відкритої кнопки.
Один безперервний рух: тягнеш ліворуч, червона зона росте разом з пальцем
і темніє після проходження порогу, відпускаєш після порогу — видаляється
одразу. Не дотягнув — плавно повертається на місце. Ніякого додаткового
тапу на кнопку після свайпу.

**Кнопка "Пропустити" в архівації перейменована на "Назад"** — тепер це
явно скасування дії (закриває форму, книга лишається в активному списку),
а не "архівувати без рецензії", щоб не плутати з видаленням.

**Кнопка підтвердження зміни імені персонажа переписана** — замість іконки
з шрифту Tabler (яка могла не рендеритись коректно) тепер кругла кнопка
з юнікод-символом "✓", не залежна від завантаження зовнішніх ресурсів.

**Той самий стиль кнопки підтвердження застосовано до редагування
характеристики та подій** — замість текстових кнопок "Скасувати"/"Зберегти"
тепер компактні округлі кнопки "✕"/"✓", візуально консистентні з полем
імені персонажа.

## Версія 1.4 — попередні виправлення

**Свайп-видалення тепер двоступеневе:** короткий свайп (як раніше) фіксує
картку відкритою з червоною кнопкою, яку треба тапнути окремо. Довгий
свайп (значно далі ширини кнопки) видаляє ОДРАЗУ при відпусканні пальця,
без додаткового тапу — червона зона візуально темніє і розтягується разом
з карткою, показуючи наближення до порогу видалення. Стосується списку
книг і списку персонажів.

**Архів — тепер окрема вкладка знизу** (Книги · Архів · Налаштування),
а не іконка в шапці. Після натискання "Дочитано" і заповнення/пропуску
форми рецензії — з'являється коротке підтвердження "перенесено в архів"
з можливістю одразу туди перейти, щоб не виглядало як видалення книги.

**Виправлено кнопку підтвердження зміни імені персонажа** — на вузьких
екранах поле вводу могло виштовхувати кнопку-галочку за межі видимої
області (той самий клас CSS-проблем, що раніше ховав кнопку видалення).
Тепер поле вводу і кнопка мають гарантовані пропорції ширини.

**Перевірено редагування подій персонажа** — підключення до поля `events`
(а не `description`) підтверджено коректним на рівні коду.

## Версія 1.3 — попередні фічі

- Редагування характеристики, подій та імені персонажа напряму (іконка олівця)
- Голосовий запис з розбором на сегменти через Gemini AI
- Лічильник згадувань, статус активності (зелений/сірий)
- Кнопка "Дочитано" → форма рецензії (рейтинг 1-10 + текст) → архів
- Тема зберігається між сесіями (Supabase)
- 4 теми оформлення

## Налаштування Supabase

Для нової бази — `supabase-schema.sql` повністю.
Якщо база вже існує — виконати по черзі: 001, 002, 003 (файли migration в репозиторії).

## Запуск локально

```bash
npm install
npm run dev
```

## Деплой на Vercel

Push у GitHub → автоматичний редеплой.

## Підключення до Telegram бота

@BotFather → `/mybots` → бот → **Bot Settings** → **Menu Button** → URL з Vercel.

## Технічний стек

- React 19 + Vite
- Supabase (база даних, включно з темою користувача)
- Gemini API через Vercel Serverless Function
- MediaRecorder API (голосовий запис)
- Власний хук `useSwipeToDelete` з двоступеневим порогом
- Іконки: Tabler Icons, шрифти: Spectral + Inter
